    # Storybook

    ## What is Storybook?
    Storybook is a development tool for creating and organizing user interface (UI) components in isolation. Imagine you are building a house and you have to design each room separately before putting them all together. Storybook works similarly, but for web and mobile apps.

    ### Main features
    - Component development: Allows you to create and test individual parts of your interface (such as buttons, menus, forms) independently, without needing to run the entire application.

    - Visual Catalog: Create a sort of "swatch book" of all the components in your app, making it easy to see what they look like and how they work.

    - Documentation: You can add descriptions and usage examples for each component, which helps other developers understand how to use them.

    - Testing and debugging: Facilitates the detection and correction of errors in specific components without affecting the rest of the application.

    - Collaboration: Improves communication between designers and developers as they can view and interact with components in a controlled environment.

    - Consistency: Helps maintain a consistent design throughout your app, as you can see all the components together and make sure they follow the same style.

## Storybook Navigation

You can see that there is a sidebar, in this you can find all the histories of your components, you can test the parameters that your components accept, you can add different variants and ensure that they are correct

## Create Story

To create stories for your components is very simple. In your project you will find a Stories folder, in this place you should create the stories of your components, creating files `Name.stories.{ts,js,tsx,jsx}`.
Choose the name carefully as it will be displayed in the sidebar.
You can also add a structure by grouping or dividing the stories by directories.

### Example

I show you a simple example of how to create a history of your component

```typescript
import type { Meta, StoryObj } from '@storybook/react';
import { YourComponent } from './YourComponent';

// ðŸ‘‡ This default export determines where your story goes in the story list
const meta: Meta<typeof YourComponent> = {
  component: YourComponent,
};

export default meta;
type Story = StoryObj<typeof YourComponent>;

//Here you can add the different variants of your component

export const FirstStory: Story = {
  args: {
    // ðŸ‘‡ The args you need here will depend on your component
  },
};
export const SecondStory: Story = {
  args: {
    // Example
    children: 'Example',
    type: 'Some',
    //...
  },
};
//...
```
